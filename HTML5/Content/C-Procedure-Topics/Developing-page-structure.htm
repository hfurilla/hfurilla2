Developing page structure for text accessibility

When creating any webpage, it’s important to properly organize your content and code to optimize readability and adhere to standards of accessibility. This is important for screen readers and those with visual and cognitive disabilities alike. Poor formatting, like large blocks of text or unnecessary text decoration, can make it difficult for users to process information on a webpage. In many cases, it leads users to abandon reading altogether. So, how do you convince users to keep reading? It all depends on a good visual hierarchy and proper heading use.

What is visual hierarchy?

Visual hierarchy refers to changes in text formatting and page structure to emphasize important information and indicate transitions between topics. Not only does visual hierarchy add to the appeal of a webpage, but it makes it far easier for users to read and understand. While some tips for developing a visual hierarchy may be merely subjective, others are crucial for accessibility.

Chunking is a method developed by UX designers that involves breaking up large paragraphs to reduce cognitive load. By sticking to 1–3 sentence paragraphs, users are granted more breaks to process information and can skim more easily. However, adding additional spacing and indents purely for appearance can disrupt those with screen readers and other kinds of assistive technology, so it’s important to add white space for readability rather than aesthetics.

Despite this, there are other ways for users to incorporate white space to provide better readability to their users. Keeping the text right aligned without any justification ensures formatting is not disrupted if the text is resized and is more pleasing to read. Using lists over commas doesn’t just assist in breaking up text but actively reinforces accessibility standards defined by the Web Content Accessibility Guidelines (WCAG). Although, the best way to build a website with visual hierarchy and accessible formatting is through proper heading usage.

Using headings for text accessibility

Within webpages, headings have dual functionality as a tool for screen readers and a way to categorize information. Heading tags add structure to a page and sort content into more concentrated areas of focus. For instance, this webpage may be about “developing page structure for text accessibility”, but the content is divided into subjects relating to this topic, such as visual hierarchy and headings.

When structuring a page with headings, you must closely adhere to the WCAG guidelines to guarantee your page is accessible to screen readers. Headings are a way for those who use this assistive technology to audibly scan the page, much like skimming. If the headings are out of order, or are only indicated through visual changes in text formatting, these pages become more dense for screen readers and harder to navigate as a result.

Heading hierarchy chart by Seobility - License: CC BY-SA 4.0.

Therefore, headings must have a rational order. HTML has six different heading tags that are available to use (h1, h2, h3, etc.). These tags must be arranged chronologically without missing information, meaning your webpage cannot start with a h2 tag then skip to a h4 tag immediately after. The heading of your page must always be a h1 tag, the next most relevant topic must be a h2 tag, subheadings within an h2 tag must be an h3 tag, and so on. If done properly, your heading tags will be neatly nested within one another, almost reminiscent of a Russian doll.

While these are standard practices outlined by WCAG, you can still style headings to have a more aesthetic appearance without impeding readability. As you may notice, the heading tags have different appearances that may sway you to skip tag levels to achieve a more desirable look. With CSS, a language that acts as a stylesheet for HTML, you can change the appearance of headers to maintain consistency while working with screen readers. While it is important to verify changes like font size, color, or other text decorations remain accessible, it’s still possible to create a page that has visual appeal without sacrificing text accessibility.
